Definición del Problema

En una cafetería se instala un robot que atenderá a los clientes. Se desea que el robot tenga iniciativa al atender al cliente, y recomiende productos antes de que se lo soliciten. El robot cuenta con un sistema de visión que le permite calcular la edad aproximada del cliente y cuenta con acceso a la web para saber la temperatura actual. A partir de la edad del cliente, y de la temperatura del día, el robot le recomendará una bebida fría o una bebida caliente.

¿Cómo va a tomar esta decisión el robot? Es aquí donde entra la probabilidad y estadística. El robot contará con datos previos sobre compras realizadas. En cada compra se registró la edad del cliente, la temperatura del día, y el tipo de bebida que compró (fría o caliente). Estos datos se encuentran en un archivo de texto y servirán para que el robot aprenda lo que debe hacer cuando llegue un nuevo cliente. Este aprendizaje lo haremos con un algoritmo bastante sencillo que se conoce como clasificador ingenuo de Bayes, y que consiste en calcular probabilidades condicionales con ayuda de la regla de Bayes.

7.- Este último punto es el elemento central del programa (y el que determina si tu proyecto tiene éxito o no).

El programa pregunta al usuario los datos del nuevo cliente (ya categorizados), y debe decidir qué tipo de bebida recomendar. La interfaz debe ser OBLIGATORIAMENTE de esta forma (lo que está en azul es lo que escribe la computadora, y lo que está en rojo es lo que escribe el usuario).

Por supuesto, que la sugerencia es resultado de aplicar un clasificador ingenuo de Bayes.

El cálculo en este último paso es bastante sencillo. Suponiendo los datos del ejemplo, se tiene que X=mayor, Y=frío.

Utilizando el teorema de Bayes se calculan dos resultados:

a) P (Z=fría | X=mayor, Y=frío)

b) P (Z=caliente |X=mayor, Y=frío)

Si a>b, entonces el robot debe ofrecer la bebida fría, de lo contrario deberá ofrecer la bebida caliente. Esta decisión en la que se elige al máximo es lo que se conoce como clasificador de Bayes ingenuo.
